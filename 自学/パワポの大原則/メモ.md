# 大原則
- パワポは説明資料とプレゼン資料を作成することができる。説明資料を作ることができれば、プレゼン資料を作成するのは容易。
- 「人を動かす」「一人歩きする」資料を早く作ることが重要である。
- スライド構成はエクセルで整理する。
- スライドタイトルは「短くまとめる」「主張しない」「体言止め、または名詞で終わる」「主語を明確にする」を守る。
- 最初のスライドから最後のスライドまで、スライドメッセージだけを読んで意味が通じるものにする必要がある。
- スライドレイアウトを編集する際は、デフォルトを使用することはないため、タイトルスライドとコンテンツスライド以外を消してしまう。
- スライドの使用範囲を「ガイド」で明示することで、作業が楽になる。
- 日本語フォントはMeiryo、英語フォントはArialにするために、スライドマスター上で変更する。
- 矢印の太さは2.25で色はグレーにする。三角矢印は二等辺三角形で作成する。
- 具体は「四角」・抽象は「楕円」で表現する。
- 直角の四角形と角丸の四角形は同じスライドに同居させない。
- 角丸の四角形も丸みは四角形の大きさを変更すると自動的に変更されてしまうため、最後にそろえる。
- 図形・表の余白は「最小化」する。具体的には左右余白を0.01cm、上下余白を0.13cmに設定する。
- 「既定の図形」で書式を自動反映する。テキストボックス、図形、線に対して有効である。
- 箇条書きは必ず1文構成となるため、句点「。」を使用しない。
- 修飾する言葉と就職される言葉を近づける。
    - ×：積極的にアメリカに事業展開する
    - 〇：アメリカに積極的に事業展開する
- グラフのルール
    - 目盛線は削除する。
    - データラベルはグラフの外側に表記する。
    - データラベルと軸目盛のフォントサイズおよび桁数を揃える。
    - 軸目盛を揃えて複数のグラフを比較する。
    - デフォルトの凡例は使用せず、テキストボックスで作り直す。その際、グラフと凡例の対応関係を明確に示す。
    - 軸目盛は細かくせず、載せる数字は5つ程度にとどめて、細かい情報はデータラベルに任せる。
- グラフは必ずパワポ上で作成する。元データのエクセルから数字データをコピペすることで対応する。

# ショートカット
|操作|意味|
|---|---|
|Ctrl+Shift+C|書式をコピー|
|Ctrl+Shift+V|書式の貼り付け|
|Ctrl+M|新規スライドの挿入|
|Ctrl+H|置換|
|Ctrl+E|中央揃え|
|Ctrl+L|左揃え|
|Ctrl+R|右揃え|
|Ctrl+[|フォントサイズの縮小|
|Ctrl+]|フォントサイズの拡大|
|Ctrl+Shift+G|グループ化の解除|
|Ctrl+矢印|図形を細かく移動|
|Alt+矢印|図形を回転|
|Shift+ドラッグ|線をまっすぐ引く|
|Alt+Shift+矢印|箇条書き順番入れ替え|
|Shift+F5|現在のスライドからプレゼン開始|
|プレゼン中にG|スライド一覧表示|
|プレゼン中にCtrl+L|レーザーポインター|
